# 富文本编辑器功能增强计划

## Context

当前项目使用 Tiptap 3 作为富文本编辑器，已有基础格式化功能（标题、加粗、斜体等），但存在两个主要问题：

1. **图片只能通过 URL 插入**：工具栏使用 `window.prompt('输入图片 URL:')`，不支持本地文件选择、拖放、粘贴
2. **格式化功能不全**：缺少表格、文字颜色、水平分割线、清除格式等常用能力

需要补齐这些短板，让编辑器达到可用的完整程度。

## 改动方案

### 1. 安装新依赖

```bash
npm install @tiptap/extension-text-style @tiptap/extension-color @tiptap/extension-table @tiptap/extension-table-row @tiptap/extension-table-cell @tiptap/extension-table-header @tiptap/extension-superscript @tiptap/extension-subscript
```

`@tiptap/extension-horizontal-rule` 已包含在 StarterKit 中，无需安装。

### 2. 新增文件

| 文件 | 职责 |
|---|---|
| `src/main/services/editor-image.service.ts` | 图片存储服务：将图片 Buffer 写入 `{storageRoot}/editor-images/{itemId}/{uuid}.ext`，返回 `file://` 路径 |
| `src/main/ipc/editor.ipc.ts` | 两个 IPC 通道：`editor:selectImage`(打开文件对话框) 和 `editor:saveImageBuffer`(接收拖放/粘贴的图片) |

### 3. 修改文件

| 文件 | 变更 |
|---|---|
| `src/main/ipc/index.ts` | 注册 `registerEditorHandlers()` |
| `src/preload/index.ts` | 暴露 `window.api.editor.selectImage()` / `window.api.editor.saveImageBuffer()` |
| `src/preload/types.ts` | 添加 `EditorApi` 接口和 `ElectronApi` 类型扩展 |
| `src/renderer/src/lib/api.ts` | 添加 `editorApi` 封装 |
| `src/renderer/src/components/editor/TipTapEditor.tsx` | 见下方详细说明 |
| `src/renderer/src/components/editor/EditorToolbar.tsx` | 见下方详细说明 |
| `src/renderer/src/views/ItemDetailView.tsx` | 向 `<TipTapEditor>` 传递 `itemId` |

### 4. TipTapEditor.tsx 变更

**新增 Props**：`itemId?: number`

**新增扩展注册**：
- `TextStyle` + `Color` — 文字颜色支持
- `Table` + `TableRow` + `TableCell` + `TableHeader` — 表格支持（可调整行列、可选表头）
- `Superscript` + `Subscript` — 上标/下标文字

**拖放/粘贴图片**：在 `editorProps` 中配置 `handleDrop` 和 `handlePaste` 拦截图片文件，通过 `FileReader` 读取为 ArrayBuffer，调用 `window.api.editor.saveImageBuffer()` 保存到本地，再调用 `setImage()` 插入。

**HTML 源码编辑**：通过 props 接收 `onHtmlEdit?: (html: string) => void` 回调，编辑器外部（ItemDetailView）控制弹窗显示。弹窗内含一个 `<textarea>` 显示当前 HTML，用户编辑后确认可替换编辑器内容。此功能不需要 Tiptap 扩展支持。

**编辑器宽度切换**：通过 props 接收 `widthMode?: 'narrow' | 'wide' | 'full'` 参数，组件外层容器根据模式切换 CSS class（`max-w-3xl`/`max-w-5xl`/`max-w-full`）。工具栏增加切换按钮。

### 5. EditorToolbar.tsx 变更

**新增按钮**：

| 按钮 | 图标 | 行为 |
|---|---|---|
| 文字颜色 | Palette | 点击弹出颜色选择面板（10 种预设色 + 原生 `input[type=color]`），选中后 `setColor(color)` |
| 表格 | Table | 直接插入 3×3 默认表格（`insertTable({ rows: 3, cols: 3, withHeaderRow: true })`） |
| 水平分割线 | SeparatorHorizontal | `setHorizontalRule()` |
| 清除格式 | RemoveFormatting | `clearNodes().unsetAllMarks()` |
| 上标 | Superscript | `toggleSuperscript()` |
| 下标 | Subscript | `toggleSubscript()` |
| Emoji | SmilePlus | 点击弹出 Emoji 面板（常见 Emoji + 搜索），选择后 `insertContent(emoji)` |
| 切换宽度 | Maximize/Minimize | 在 narrow/wide/full 三种宽度模式间循环切换 |

**修改图片按钮**：从 `window.prompt()` 改为调用 `window.api.editor.selectImage(itemId)`，然后在编辑器中 `setImage({ src: result.localSrc })`。

**HTML 源码编辑按钮**：在工具栏增加一个 `Code/FileCode` 图标按钮，点击后通知父组件（ItemDetailView）打开 HTML 源码编辑弹窗。弹窗内用 `<textarea>` 展示当前 `content_html`，确认后回写编辑器内容（`editor.commands.setContent(html)`）。

**工具栏布局**：用分隔线分组为 8 组：
```
撤销/重做 → 加粗/斜体/下划线/删除线/行内代码/上标/下标 → 颜色/清除格式 →
H1/H2/H3 → 无序列表/有序列表/任务列表 → 引用/代码块/分割线 →
左对齐/居中/右对齐 → 表格/链接/图片/Emoji → 切换宽度/HTML源码
```

### 6. 图片存储策略

```
{storageRoot}/editor-images/{itemId}/{uuid}.{ext}
```

- 路径基于 `config.ts:getStorageRootPath()`
- 文件名用 `uuid` 生成，避免重名冲突
- HTML 中引用 `file:///D:/KnowledgeBase/editor-images/123/uuid.png` 格式
- 首次不写入 `attachments` 表，图片在 HTML 内容中直接引用

### 7. 数据流

**工具栏点击图片**：点击 → IPC(`editor:selectImage`) → Electron 文件对话框 → 读取文件 → 写入本地存储 → 返回 `file://` 路径 → `setImage()` 插入编辑器

**拖放/粘贴图片**：拖放/粘贴 → 编辑器拦截事件 → `FileReader` 读取 ArrayBuffer → IPC(`editor:saveImageBuffer`) → 写入本地存储 → 返回 `file://` 路径 → `setImage()` 插入编辑器

## 验证方式

1. `npm run dev` 启动应用
2. 打开任意 note/article/bookmark 类型条目，点击编辑
3. 验证新增格式化按钮：文字颜色（弹出颜色面板选择）、表格（插入 3×3 表格可调整行列）、分割线、清除格式、上标、下标
4. 验证 Emoji 选择器：点击 Emoji 按钮弹出面板，可选择 Emoji 插入
5. 验证 HTML 源码编辑：点击 HTML 源码按钮 → 弹窗显示编辑器 HTML → 修改内容确认 → 编辑器内容同步更新
6. 验证宽度切换：点击宽度切换按钮 → 编辑器容器在 narrow/wide/full 三种宽度间切换
7. 验证图片导入：点击图片按钮 → 弹出系统文件对话框 → 选择本地图片 → 图片显示在编辑器中
8. 验证拖放：从文件管理器拖拽图片到编辑器区域 → 图片保存并显示
9. 验证粘贴：截图工具截取后 Ctrl+V 粘贴到编辑器 → 图片保存并显示
10. 验证工具栏布局：分组清晰，所有按钮正常触发
11. 验证保存：修改后保存 → 刷新页面 → 图片和格式全部保持
